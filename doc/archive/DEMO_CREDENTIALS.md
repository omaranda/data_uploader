## Security Checklist for Production

Before deploying to production:

- [ ] **Delete all demo data** or change all passwords
- [ ] Generate strong unique passwords for each user
- [ ] Update the default admin password (`admin` / `admin123`)
- [ ] Remove or comment out `04_demo_data.sql` from docker-compose.yml
- [ ] Use environment variables for sensitive data
- [ ] Enable HTTPS/TLS for all connections
- [ ] Set up proper AWS IAM roles with minimal permissions
- [ ] Configure firewall rules to restrict database access
- [ ] Enable database encryption at rest
- [ ] Set up automated password rotation
- [ ] Implement MFA for admin accounts
- [ ] Regular security audits and penetration testing

## Generating New Password Hashes

To generate a hash for a new password:

```bash
cd backend
./venv/bin/python3 ../scripts/generate_password_hash.py
```

Or use the backend API after deployment:
```python
from app.utils.security import hash_password
new_hash = hash_password("your_new_password")
print(new_hash)
```
