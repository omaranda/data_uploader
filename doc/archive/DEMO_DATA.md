## Loading Demo Data

Demo data is automatically loaded when you start the Docker stack:

```bash
docker-compose up -d
```

The SQL scripts execute in this order:
1. `01_init.sql` - Base schema
2. `02_migration.sql` - Multi-tenancy migration
3. `03_seed_data.sql` - Default admin user
4. `04_demo_data.sql` - Demo companies, users, and projects

To reload demo data (this will **delete all data**):
```bash
docker-compose down -v
docker-compose up -d
```
